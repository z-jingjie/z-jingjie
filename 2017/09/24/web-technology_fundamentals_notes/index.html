Introduction (Web技术概览)

1. Core Basic: Hosting, Serving, Rendering

2. Internet vs. Web

Web is a type of Internet, transfer data via HTTP (the standard protocal for this task)

3. Client vs. Server

client.send(URL) –> DNS_Server.translate –> client.received(IP) –> client.send(HTTP Request)
case 1: –> server.send(HTTP Response: HTML) –> client.render(HTML)
case 2: --> server.send(CSS, IMG, JS, etc) –> client.received(resources) –> client.render(resources _documents)

4. Static Site Pages vs. Dynamic Site Pages

Static Site Pages: No additional processing
Dynamic Site Pages: further processing operated by Web Application (like PHP, .Net or Ruby), then data sent to client for rendering, and stylesheet/script/img are then executed

5. DNS matters a lot

local ISP DNS server, root name server, top-level domain
client.send(hostname) –> local ISP DNS server –> root name server –> .com name server –> client.sendrequestto(.com name server) –> IP
DNS caching, takes up to 24 hours

6. Internet Protocals (referred to “The Internet Protocal Suite”)

TCP/IP: since 1973, used for data exchanging
Multiple Layers of Protocals to handle different tasks:

  • Application Layer: HTTP, FTP, SMTP, POP, DNS, RTP, RIP, SNMP, … (create or share data over the web)
  • Link Layer: Wireless LAN, Ethernet, ATM, … (how each other connected over a networ21k)
  • Transport Layer: TCP, UDP, … (how ports are assigned for communication)
  • Internet Layer: IPv4, IPv6, … (addressing & arouning structure for data across the network)

7. HTTP & HTTPS (layered with a security protocal)

FTP: connect to file server instead of web server, used for large file transfering
　　 once the HTTP Request has been sent and received, it would be discarded by both side (lightweight, efficient)
　　 Persistent data for browsing sessions need alternative methods like: Cookies, JavaScript Vars, Server Session Management
Cookies: small text file that the page write on the client machine

8. URL Anatomy (URL结构剖析)

Example: http://www.google.com/map
　　　　　protocal + resource name (subdomain & domain & top-level domain) + port + path
　　　　　SubDomain: a folder located on the web server (example: site root > www)

9. How Browsers Work

Data: HTTP
Browser Components:
1. Browser UI (Visual chrome, Address bar, Other visual elements)
2. Internal Engine (directs communication across various components; manage entire process)
3. Rendering Engine__ (Parses HTML & CSS; render the content) [最显著的浏览器表现区别通常来自于此]
　 ① IE, Windows Mobile - Trident
　 ② Firefox Series: Gecko
　 ③ Opera Series: Presto
　 ④ Safari, webOS, Android: Webkit
　 ⑤ Chrome, Opera 14+, Android 4.4+: Blink
4. Networking (to communicate through HTTP and other protocal components)
5. Data Storage (internal data storage; session management)
6. JavaScript Interpreter (execute client-side scripts)

10. Web Standards & the W3C

Mid 1990s: Netscape vs. Microsoft
1994:　　　W3C founded (issues recommendations for web technologies)
1998:　　　Web Standards Project (refers to W3C recommendations as Standards)
2013:　　　claimed that web standard project’s work has done

11. Web Server basics

  • a computer with softwares or components
  • signed in IP address, registered through DNS server
  • uses virtual hosting (one IP, multiple sites) [shared server vs. dedicated server]
  • common file structure: Root > images + css + scripts + *.htm (+ database, …)
  • Web Stacks: groups of software that work together to build and process the websites
    Common web stack components:
    　Operating System + Webserver + Database Server + Programming Language
    Examples:
    ① LAMP: Linux + Apache + MySQL + PHP(or Pro, Python) [All open sourced]
    ② WAMP: Windows, Apache, MySQL, PHP
    ③ WISA: Windows, IIS, Ms SQL, .NET
    ④ MARS: Solaris, Apache, MySQL, Ruby

Front-End Technology (前端技术)

1. Front-end Design

Front-end designer & Front-end developer mean the same thing in fact
Both: HTML, HTML Prototypes, CSS Preprocessor, JavaScript, jQuery, gunt.js, Github
Front-end designer (More UI-base): CSS, UX Design, Image Editing, Sketching
Front-end developer: AJAX, Node.js, IndexDB, GIT, templating languages
Typical Front-end Designer Roles:

  • Creating mockups & developing visual standards (视觉建模)
  • Structuring semantic content (语义化)
  • Handling site accessibility (可访问性)
  • Site typography (字体样式)
  • Page layout (页面布局)
  • Form design (表单设计)
  • Interactivity & animation (交互 & 动画)
  • Creating site imagery (站点形象化)
  • …

2. HTML, XHTML, HTML5 History & Structure

3. CSS: external styles; embedded styles; inline styles

selector: {declaration} declaration: property: value;
Benefits of CSS:

  • Allows for modular site design (站点设计模块化)
  • Faster development by centralization of styles (样式集中处理，加速代码部署)
  • Create portable and reusable styles (样式便携性 & 代码复用性)

4. JavaScript

JavaScript Functionality:

  • Allows designers to create interactive elements (打造可交互的页面元素)
  • Can open new windows or give users feedback
  • Can update, sort, present data directly in the browser 在浏览器内(更新、排序、展示数据)
  • No sever interaction is required (无需服务器介入即可运转)

Approaching JavaScript Advice:

  • Learn the basics of JavaScript FIRST
  • Experiment with popular Framwork
  • When JavaScript is not enabled, have strategy for providing fallback content

Common Image Concerns:

  • Understand the IMAGE’s operational principle within browsers
  • Find a balance between file size and image quality (bandwidth issue)
  • Image is especially important with the rise of mobile devices with hi-res displays nowadays
  • Use the formats that browsers support

JPG/JPEG (Joint Photographic Experts Group):

  • Standard since 1992
  • Among the most widely used graphic formats in existence
  • Use lossy compression, will cause compression artifact (压缩失真) in high levels of compression
  • Doesn’t support transparency

GIF (Graphics Interchange Format):

  • Since 1987
  • Among the most widely used graphic formats in existence, for icons & images with solid regions of color
  • 8-bit graphics, 256 colors limited, support transparency & limited animation

PNG (Portable Network Graphics):

  • Since 1995 [Open Source]
  • Support transparency, use lossless compression format
  • Support 16,777,216 colors, file size is relatively larger

SVG (Scalable Vertor Graphics):

  • Vector graphics format
  • Can be scaled independently of resolution, without quality loss
  • Written in SVG markup, editable through codes like XML or HTML, can be further styled through CSS
  • Widely supported among devices

5. API (Application Programming Interface)

A set of instructions and standards for accessing services or components of a specific system
API Considerations:

  • Most APIs limit the available amount of data or types of services
  • APIs change, developers need to ensure the services in use are monitored
  • Have a backup plan or fallbacks in case the APIs fail to work

6. HTML5 APIs

WHATWG develops it, W3C supports it later
W3C took several APIs out of the HTML5 specification and moved them into W3C’s own specifications, in order to allow the APIs mature independently and reduce the size of HTML5 specification
Media API: control the loading and playback of media, related to the Text Track API that control captions and subtitles
Drag and Drop API: allow elements to be dragged & dropped from inside or outside the browser
App Cache API: store data offline, make offline persistent applications possible, sync the data when connections restored
Geolocation API: standalone, associated with HTML5, report the current location of users
Canvas API: standalone, associated with HTML5, create canvas area in browser for drawing/painting, help creating complex games/animations/visual effects without plugins (Flash, etc …)

7. Web Fonts

client.send(web-font-request) –> client.receive(web-font) –> client.install(web-font) –> css(@font-face)
redesign or modify needed: reducing glyphs, creating screen-specific hinting, new licenses for web fonts using
allow for downloading and temporary installationof fonts within browser
mostly used to describe the CSS @font-face technique
How to host: hosting services like Typekit, self-host if licensed
Web Fonts Types: EOT (Embedded OpenType), WOFF (Web Open Font Formast)

Back-End Technology (后端技术)

1. Server-side Scripting

early days: CGI scripts [C++, Perl]
later: PHP, Java, .Net, CFML
Server-side script processing: database I/O, calculation, select displaying type of content

Concerns:

  • done on the server side, will increase server load of high traffic sites
  • usually requires a page refresh for data processing [the reason why client-side scripts popular]
  • Ajax/other client-side bridges help enhance the usabilityu of server-side scripting

2. PHP’s role

Since 1994 [Personal Home Page], developed in 1997 as PHP 3.0 [Hypertext Preprocessor]
Important because it paired with Linux, Apache, MySQL …
.php, .asp, .aspx, .cfm: process on the server before running in browsers, code is invisible for end-users
example: [it is in fact a .html file]
软件：*MAMP (manage your website locally)

3. Other popular server-side language

JSP (Java Server Pages): part of the Java Framework, scripts are combinations of XML and Java scriplets
.NET (Visual Basic .NET): sometimes referred to .NET as VB .NET, which is in fact a part of .NET Framework
　　　　　　　　　　　　 often used for large-scale sites or applications, hosted on Windows servers only
ColdFusion: using its own markup language CFML, central admin simplify the tasks, require less code
　　　　　　hosts are hard to find, and costly than others
Python: powerful, has a variety of web-related libraries for building sites and web-based applications
　　　　often used for large-scale sites or applications
Ruby: a general-purpose development language, became popular due to Ruby-on-Rails Web Framework

4. Selecting a Server-side Language [Platform-Choosing]

  • Cost
  • Developer Network (Community?)
  • Ease of use
  • Ask developer for assistance & Do research for info

5. Database

Factors that influence the ultimate database storage scheme: …
Cookies是网站服务器向浏览器发送的一小段文本，记录了用户的访问活动信息
Cookies [security concern] –> Web Storage API [HTML5]: * / XML files / JSON files
Two main scheme that allows Database be stored within the browser: Indexed DB vs. Web SQL
Two main types** of online database:

  • Relational Databases: table <-- item key values –> table (allows for complex sorting and filtering)
    　　　　driven by a schema, would be difficult to modify after creation
    　　　　　　　　　　　Oracle, MySQL, Sybase, other SQL-based databases
  • Non-Relational Databases: NoSQL databases store data individual flat, object-driven datasets
    　　　　often faster to index, and scale easier than relational databases
    　　　　　　　　　　　　　　MongoDB, CouchDB, SimpleDB, HBase, CouchBase, Cassandra, …

6. SQL (Structured Query Language)

standard language for managing and retrieving info from databases [CRUD操作]
SQL syntax: select, insert, update, delete
Example: SELECT FROM tours WHERE price < 200 ORDER BY tourName ASC
　　　　　(select all data, from tours table, pick price < 200, sort in ascending order by tourName)
　　　　　_results will return as a table, often referred as a *dataset_

Learning SQL:

  • SQL has multiple versions, with different extensions
  • SQL queries usually written as a string, inside the web application code

7. CMS (Content Management System)

a application for content creation, management, publishing, and archiving

CMS Benefits:

  • updating content through a WYSIWYG (What you see is what you get) editor, which is easy to use
  • users can create user groups with different permission levels
  • users can create content/pages/regions within the site without involving a designer
  • users can add plugins/extensions to the sites without the need of developers or additional skills
  • Exploring CMS options:
  • hosted online, only require an account to start building the hosted sites
  • other CMS can be installed on self-owned web server
  • features and page customization differ between CMS, be sure to research the info at first [source code]
  • Mainstream CMS: Joomla, Drupal, Movable Type, ModX, Radiant, Wordpress, Expression Engine, …

8. CDN (Content Delivery Network)

a system for delivering content over a distributed network of servers
reduce bandwidth to self-hosted server, speed up the site loading, protect server from DDoS attack, …
largely used to serve static resources, like javascript libraries, media files, site dependencies

Types of CDNs:

  • host a single type of resource: Google Hosted Libraries, Boostrap CDN
  • serve as delivery networks(分发网络), also as content aggregators(内容聚合器): Youtube
  • Mainstream CDNs: Akamai, CloudFlare, Amazon’s CloudFront, EdgeCast, Windows Azure, …

CDN Issues:

  • adds an additional layer of complexity to the site [small/low-traffic sites might not need it]
  • must load the entire resources, can’t create customized/minified libraries/frameworks on CDN
  • an external CDN might bring security risks to the site [can’t control the site content completely]

9. Cloud Services

Cloud Computing: usually refers to distributed processes over the internet
　　　　　　　　　 (一个借助网络规模和计算能力，来处理工作任务的计算机网络)

Mainstream Cloud Services: Google Cloud Storage, Amazon Web Services, Microsoft Cloud Services, Heroku, Rackspace, …
云服务/云应用的崛起，正剧烈地变革着互联网中内容信息的分发模式，互联网不再仅仅只是一个承载着万维网的硬件网络，而且还是一个稳健且成熟的大规模联网应用平台

10. GitHub, from Git (important in web development process)

an online distribution services (with web interface), allows to store/version-control/share repositories
built around Git (command line tool), which is an open source version control system

Assembly Technologies (封装技术)

1. JavaScript Libraries (JS函数库)

collections of pre-written JavaScript methods & functions
ranging from small focused sets like Date.js, to broader task-oriented libararies like jQuery

Pros of using libraries:

  • more efficient to write JavaScript
  • extend coding capabilities
  • code is widely used and tested, consistently improving by contributors

Cons of using libraries:

  • increase the size of code, add additional requests
  • tied to specific library syntax
  • native JavaScript unknown, not always sure if the library code is more appropriate
    [developers need to investigate the options, to make sure the code is efficient enough]

2. Frameworks (框架) & Boilerplates (模板)

collections of prebuilt HTML & CSS & JavaScript package, for speeding up the site development
Boilerplate: a set of templates(模板) built around a specific starting point(standard) or goal
　　　　　　provide a starting point for building sites & apps
Framework: a collection of assets(自主开发的静态资源素材, 如image, css, js等) help building sites or apps
　　　　　　usually has specific focus, like CSS grids, JavaScript libraries, JavaScript scripts, HTML templates

most frameworks are the result of studio or an individual creating a starting point and system that allow them to build their site a little bit more efficient (大多数框架是工作室或个人为了更高效地开发网站而打造的初始模板和系统的结果)

Complex UI Frameworks:

  • Framework like Bootstrap(for CSS) are built around developing the front-end of sites or apps
  • UI Frameworks are typically fully functional development tools (这种包含了站点各方面的框架构建)
  • add the most overhead designs, also provide widest set of tools to complete the front-end of project

Minimal Frameworks:

  • typically have a singular goal or focus (这种仅侧重站点某一方面的框架构建)
  • designed to have a small footprint, and help out in specific areas
  • the suitable framework will be largely dependent on personal philosophy and project type

Prototyping(打造站点原型):

  • oftenly used to build HTML prototypes
  • dramatically speed up prototype development and iteration (加速建站过程)
  • in most cases, framework code is NOT used for final production
    (tie users to its structure, methods, overhead designs)
    注：static (第三方开发的静态资源素材, 如jQuery.js, Bootstrap.css等)

3. CSS Preprocessors [CSS预处理器所输出的样式代码正逐步转换为CSS原型代码形式，可预见的将来这种工具或将被抛弃]

a scripting language that extends the functionality of CSS with new syntax, and must be compiled into native/regular CSS code before publishing/running through processor
Two-Step Process: using preprocessor syntax –> compile the file to generate the actual *.css for site

Pros of Preprocessor:

  • add additional functionality that CSS lacks, like variable, conditional logic
  • the syntax makes CSS writing more efficient
  • dealing with many browser inconsistencies automatically when processing CSS

Cons of Preprocessor:

  • teammates need to use the same preprocessor (syntax) to share/reuse the production code
  • create more code/size than manual CSS, may cause unneccesary code bloating and inefficient
  • additional layer of complexity to workflows

Popular CSS Preprocessors:

  • Sass (written in Ruby): two versions of syntaxes for choosing (有两种语法版本)
  • Less (written in JavaScript): syntax closely resembles native/regular CSS [easy for code transitioning]
  • Stylus (written in JavaScript): very minimal/stripped-down syntax (语法简洁)
  • PostCSS
