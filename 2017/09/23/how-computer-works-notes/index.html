《计算机是怎样跑起来的》读书笔记

notes
 2017/09/23

01. 三大原则

  1. 硬件上看，计算机是执行输入、运算、输出的机器
  2. 软件上看，程序是指令和数据的集合
  3. 计算机的处理方式有时与人们的思维习惯不同，计算机用数字表示一切数据

02. 制造一台计算机

主要组成部分：CPU、内存(DRAM)、PI/O、时钟发生器、电阻、电容、开关、电源
电源：Vcc引脚，+5V，”1”；GND引脚，0V，”0”
IC引脚：正方向标志朝左，逆时针编序号、侧边写作用代号
寄存器：CPU中的，存储数据+数据运算：累加器A、标志寄存器Flag、程序指针PC、栈顶指针SP
​ I/O中的，存储数据(数据寄存器) + 设定I/O功能(控制寄存器)
电阻元器件颜色代码：黑灵芝、粽子叶、红孩儿、三乘轿、黄丝带、五缕须、蓝琉璃、 钟子期、灰八哥、摆酒宴
​ 色环靠近接头的一侧朝左，银色代表的误 差等级是±10%，金色代表的误差 等级是±5%

03. 体验手工汇编(Hand Assemble)

“助记符(memonic)”语法：标签(地址又名，如”LOOP:”)、操作码(指令)、操作数(指令对象，即寄存器名字、地址或数据)

存储顺序：大端序存储(Big Endian)， 高位地址在前→低位地址在后
​ 小端序存储(Little Endian)，低位地址在前→高位地址在后
估算程序的执行时间：指令数、时钟周期数、时钟信号频率

04. 程序的流动

程序的流程：顺序执行、条件分支、循环(流程图中：帽子和短裤)
作为程序员必须要学会灵活地运用流程图。在思考程序流程的时候，也要首先在头脑中画出流程图。
结构化程序设计(一种编程风格)：在高级编程语言层面，仅使用顺序执行、条件分支、循环语句，以程序块表示程序流程
特殊程序流程——中断处理：程序流程突然跳转到特定位置(“中断处理例程Routine”、又称”中断处理程序Handler”)
​ CPU中INT和NMI两个引脚，可以接收I/O设备发出的中断请求信号，然后转入中断处理流程
特殊程序流程——事件驱动(Event Driven)：视鼠标/键盘操作为事件，OS检测事件的发生，
​ [GUI环境，常用”状态转化图”代替流程图] 并调用WndProc()函数通知应用程序，应用程序随后进行相应的程序处理
状态转化图(表)：反映了应用程序由于某种原因从某个状态转化到另一个状态的流程

05. 算法七大要点

算法(解决问题的具体而有限的步骤)：被明确定义的有限个规则的集合，用于根据有限的步骤解决问题
思考算法两步走：整体考虑大致流程(初始化处理→循环处理→收尾处理) + 具体考虑细节流程
要点1：算法中解决问题的步骤是明确且有限的
要点2：计算机不靠直觉而是机械地解决问题
要点3：了解并应用典型算法：算法工具书 & 个人思考算法的习惯 《Algorithms + Data Structures = Programs》
​ 辗转相除法(最大公约数)、埃拉托斯特尼筛法(素数?)、顺序查找、二分查找、哈希查找、冒泡排序、快速排序
要点4：利用计算机的处理速度
要点5：使用编程技巧提升程序执行速度：改善时空复杂度，使用”哨兵”(常用于线性搜索，如取目标值并预先设置在最后)
要点6：找出数字间的规律，简化处理流程
要点7：先在纸上考虑算法：先构建文字、图表，再代入数据

06. 数据结构七大要点

数据结构：作为被处理对象的数据的排列方式
要点1：了解内存和变量的关系：编程语言使用变量，把数据存入内存空间里
要点2：了解作为数据结构基础的数组：连续分布，反映出内存的物理结构，所以是数据结构的基础
要点3：了解数组的应用——作为典型算法的数据结构
要点4：了解并掌握典型数据结构的类型和概念: 

1. 栈 - 把数据堆积起来 
2. 队列 - 把数据排成一队
3. 链表 - 改变数据的排序
4. 二叉树数据结构多用于搜索数据的算法中 二叉树 - 把数据两路排列，是链表的特殊形态(指针数量≥2)

要点5：了解栈和队列的实现方法：栈顶指针、栈大小；队首指针、队尾指针、队列长度
要点6：了解结构体的组成：struct str_name{type name;}str_name, str_name; 要点7：**了解链表和二叉树的实现方法：指针pointer

07. OOP(面向对象编程)思想 [七种解释]

面向过程编程思想：自顶而下、逐步细化
①将要实现的功能描述为一个由始至终、按部就班地连接的步骤
②依次实现这些步骤，若步骤难度较大，可以继续细化为若干子步骤，依此类推
③程序的主体是函数，一个函数计算一个封装起来的模块，以实现特定的功能，各个子步骤也往往是通过调用函数来完成的，从而实现代码的重用和模块化编程

面向过程 vs 面向对象：
①都支持复用和模块化，但面向对象封装程度更深、更封闭、更安全
②面向对象的思维方式更接近于现实生活，更适合解决大型、复杂项目的业务逻辑
③从前期开发角度看，面向对象比面向过程更加复杂；从维护和扩展角度看，面向对象远比面向过程简单
④面向过程的代码执行效率更高，函数调用层次较少

面向对象模拟现实世界：在现实世界中，任何一个操作或业务逻辑的发生，都需要一个实体来完成，实体就是动作的支配者。
OOA：面向对象分析
OOD：面向对象设计
OOP：面向对象编程

面向组件编程(Component Based Programming) [面向过程编程？] - 各模块(函数) >> 组装 >> 程序
面向对象编程(Object Oriented Programming) - 业务 >> 建模(构建”类”) >> 程序 [C >> C++ >> C#, Java]

  1. 进行传统设计时，先考虑构成程序的功能和数据，然后确定相应的函数和变量。
  2. 进行OOP设计时，先确定所需的类(支配操作的实体)，再在类中列出函数/方法(功能)和变量(属性)，最后封装以供调用。

OOP编程：关注点置于对象(Object)本身，其构成要素包含对象的行为及操作，以此为基础进行编程。
OOP的基本编程技巧：继承(拷贝现成类的架构)、封装(黑箱)、多态(同一个消息，不同对象有不同操作，方便记忆)
类(class)的意义：模拟现实世界中支配运动的实体，指具有相同或相似属性的一组实体的集合
对象(Object)：把指令和数据归拢到具有一定意义的组中而形成的整体(实体)
类与对象的关系：类是对象的规范，对象是类的实例(类是模具，对象是类在内存中的副本)
类的使用：调用类的个别成员、类的定义中包含其他类(组合)、继承已有类后定义新的类
观点1：OOP通过把组件拼装到一起构建程序：把程序中相关联的函数和变量汇集成类(class)，视为组件
观点2：OOP能够提升程序的开发效率和可维护性：

1. 环境中现成的类，构成"类库"
2. 通过"类"来模块化地开发、维护、扩展
3. 类的使用规范(操作语法)——接口(Interface)

观点3：OOP是适用于大型程序的开发方法：比如在一个类中，封装数十上百个函数+变量
观点4：OOP就是在为现实世界建模(分析、重现对象的组成)：组件化(模块化)、省略化(抓住重点)
观点5：OOP可以借助UML(统一建模语言)设计程序：[↙9种建模表记方法]

1. 用例图(Use Case Diagram)      表示用户使用程序的方式 
2. 类  图(Class Diagram)         表示类以及多个类之间的关系 
3. 对象图(Object Diagram)        表示对象
4. 时序图(Sequence Diagram)      从时间上关注并表示多个对象间的交互
5. 协作图(Collaboration Diagram) 从合作关系上关注并表示多个对象间的交互
6. 状态图(Statechart Diagram)    表示对象状态的变化
7. 活动图(Activity Diagram)      表示处理的流程等
8. 组件图(Component Diagram)     表示文件以及多个文件之间的关系
9. 配置图(Deployment Diagram)    表示计算机或程序的部署配置方法

观点6：OOP通过对象间的消息传递驱动程序：比如让A对象调用B对象的成员函数 (可用”时序图”和”协作图”表示)
观点7：在OOP中使用继承、封装(public/private)和多态

08. 理解数据库

DBMS(Database Management System)：数据库管理系统 (使应用程序读写数据文件，且能一致并安全地存储数据，还可检查参照完整性)
SQL(Structured Query Language)：结构化查询语言
数据库构成：应用程序 → DBMS → 数据文件
数据库系统类型：

1. 独立型系统           - 本机(数据文件+DBMS+应用程序)
2. 文件共享型系统         - 服务器(数据文件)               ←(LAN)→ 客户端(DBMS+应用程序)
3. 客户端/服务器型系统    - 服务器(数据文件+DBMS)          ←(LAN)→ 客户端(应用程序)
4. Web系统             - 服务器(数据文件+DBMS+应用程序)  ←(WAN)→ 客户端(Web浏览器)

数据库类型：阶层型(树)/网络型数据库，卡片型数据库，关系型数据库RDB，面向对象型数据库，XML型数据库(如RSS)
数据表中的横行：记录、行、元组(tuple)
数据表中的竖行：字段、列、属性(attribute)
主键(Primary Key)：在表内唯一标识一条记录的ID(字段)
外键(Foreign Key)：在表中添加的、其他表主键的字段(在表和表之间建立关系)
索引(Index，与键无关的内部机制)：DBMS中便于排序/检索的数据结构(索引表) (一般使用二叉树排序索引键值)
事务：由用户定义的一个数据库操作序列(SQL语句)，是数据的基本处理单位 (如银行转账事务)
设计数据库系统：优先设计数据库，然后设计用户界面

09. 理解TCP/IP网络

LAN(Local Area Network)：局域网
WAN(Wide Area Network)：广域网，如互联网
TCP/IP(Transmission Control Protocol/Internet Protocol)：传输控制协议/网际协议，指的是协议族
协议：对信息发送方式的规定或约束
MAC(Media Access Control)地址：烧录于网卡ROM中的设备唯一编号，由制造厂商编号和产品编号两部分组成(6 Bytes)
IPv4地址：用于对计算机分组、管理(4 Bytes)，表示分组的部分叫”网络地址”，表示主机(Host)的部分叫”主机地址”
IPv6地址：16 Bytes，十六进制表示 (地址空间更大，路由表更小[聚类原则]，加强组播和对流支持，引入自动配置机制)
v4与v6地址转换技术(过渡方案)：NAT64 + DNS64
子网掩码(Subnet Mask)：标识出32bits的IP地址中，网络地址和主机地址的各自表示范围 [以二进制的”1”和”0”区分]
DHCP(Dynamic Host Configuration Protocol)：动态主机设置协议
[DHCP服务器上记录着可以被分配到LAN内计算机的IP地址范围和子网掩码值]
集线器(Hub)：负责把各台计算机的网线相互连接在一起的集线设备
路由器(Router)：负责把LAN和WAN连接起来的设备(一般是：LAN–LAN’s Router–ISP’s Router–…–WAN)
[查看数据包头上附加的IP地址中的网络地址部分，根据路由表内的规则，决定数据包在LAN内直发，或经路由器转发至LAN外]
路由(Routing)：数据经过路由器被转发的过程
Tracert(跟踪路由)：路由跟踪诊断实用程序，将包含不同TTL值的Internet控制消息协议(ICMP)回显数据包发送到目标，
以决定到达目标采用的路由、查看转发路径
tracert命令出现”请求超时”，但仍显示下一跃点(路由)信息的可能原因：
a. 那一跳禁ICMP协议，具体是禁PING (检测目标主机是否可连通)
b. 那一跳不对TTL超时(TTL == 0 or 1)响应，直接丢弃
c. MPLS/VPN(多协议标记交换/虚拟专用网络)网络
d. 超时的节点设备宕机/关机，无法响应ICMP协议
e. 超时的节点网络拥塞或网络质量差延时过大
跃点：即路由器，数据包传输过程中所经过的网络设备点
跃点数：数据包所经过的跃点的累计值(权)，用于在路由表中选出最能匹配转发包中目标地址的路由 (数值范围1~9999)
自动跃点：
TTL(Time to Live)：存活时间，数据包经过一个网络时，可传递的最长距离(跃点数) [每经过一个跃点，其值减一]
路由表(由各LAN中的路由器相互交换、更新)：记录本路由器到相邻路由器的数据包传输路径 [cmd命令：route print]
(Network Destination)网络目标
(Netmask)网络掩码
(Gateway)网关 - 路由器在LAN中的IP地址
(Interface)接口 -
(Metric)跃点数 - 路径的权重 (路由器根据算法选择权重值较小的路径)

网卡种类：以太网(Ethernet)，…
CSMA/CD(Career Sense Multiple Access with Collision Detection)：带冲突检测的载波监听多路访问 (用于LAN)
载波监听(Career Sense)：监听网络是否正使用电信号(Career)的机制

DNS(Domain Name System):域名系统，DNS服务器常部署于各LAN中，记录着FQDN和IP地址的对应关系表，对域名进行解析
FQDN(Fully Qualified Domain Name)：完整限定域名，由主机名和域名组合的名字，如zcc.yze.co.uk
cmd命令： hostname - 查看主机名
ipconfig /all - 查看主机名+(DNS)域名后缀
nslookup - 询问DNS服务器

ARP(Address Resolution Protocol)：地址解析协议，实现由IP地址到MAC地址的转换 (有缓存功能，即ARP缓存表)
工作原理：对LAN内全部主机”广播(Broadcast)”询问，若收到某主机的MAC地址回复，则确定其IP与MAC的对应关系

IP协议：用于指定数据发送目的地的IP地址，以及通过路由器转发数据
TCP协议：用于通过数据发送者和接收者相互回应确认信号，可靠地传输数据[“握手”，数据以”包(Packet)”为单位]
数据包：MAC信息+IP信息+TCP信息+数据+错误检查信息 (层层封包 → 网卡间数据传输 → 层层解包)

10. 数据加密

信息摘要(Message Digest)：指从作为数字签名对象的文件整体中，计算出的数值 (证明文件唯一性，如MD5)
数字签名：加密处理过的信息摘要 (公开密钥加密技术)
密钥：用于加密和解密过程的数字
​ (合理的密钥应满足：长短适中、可以反复使用、可以 通过某种通信手段交给接收者，且通信双方以外的难以用密钥解密)
加密简单原理：将字符序列由”明文”变换为”密文”(加密)，然后逆向还原(“解密”)
逻辑异或运算(XOR)：对应位置上的两个二进制数，值相同则为0，值不同则为1
　对称密钥加密技术(秘密密钥加密技术)：加密、解密时密钥相同
非对称密钥加密技术(公开密钥加密技术)：RSA算法 (加密用公钥，解密用私钥，二者数值不同，创建数字签名时操作相反)
字符编码：对应各种字符的数字
字符集：字符编码体系 (ASCII字符集、JIS字符集、Shift-JIS字符集、EUC字符集、Unicode字符集等)

11. XML(Extensible Markup Language，可扩展标记语言)

标记：通过添加标记标签，为数据赋予意义的行为
标记语言：定义了标记的规则的语言 (一套标签)
SGML(Standard Generalized Markup Language，标准通用标记语言)：标记语言的始祖
CSV(Comma Separated Value，逗号分隔值)：XML、CSV都是通用的数据交换格式
HTML(Hypertext Markup Language，超文本标记语言)：用于编写网页的标记语言 (决定了可用于编写网页的标签)
XHTML(Extensible Hypertext Markup Language，可扩展超文本标记语言)：用XML定义HTML4.0
HTML5：HTML的第五次重大修改版本 (是一项推荐标准)
W3C(World Wide Web Consortium，万维网联盟)：以“W3C推荐标准”的形式制定一系列标准
DOM(Document Object Model，文档对象模型)：用于处理XML文档的程序组件
SAX(Simple API for XML)：用于处理XML文档的程序组件
XML Schema：用于定义 XML实例结构的新技术
HTML与XML区别：

1. HTML是用于编写网页的标记语言
2. XML是用于定义任意标记语言的元语言 (仅限定了标记时标签的书写格式/风格) [用XML解析器检查格式]
3. HTML的标签种类是固定的，HTML中规定的各种标签只能用来指定信息的呈现样式，而不能表示信息的含义
4. XML的标签可以扩展自定义，通过XML可以定义各种新语言，为在互联网上交换的信息赋予意义

可以为XML标签设定XML命名空间(Namespace in XML)：<标签名 xmlns=”命名空间名”>标记内容</标签名>
完整的XML文档包括：

SOAP(Simple Object Access Protocol，简单对象访问协议)：用于分布式计算 (由XML定义的一种标记语言)
分布式计算：把程序分散部署在用网络连接起来的多台计算机上，使这些计算机相互协作，充分发挥计算机整体的计算能力

计算机系统：将各种各样的硬件和软件组合起来构 建而成的系统

npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: false
    tags: true
